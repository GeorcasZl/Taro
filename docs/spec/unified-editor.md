# 统一编辑界面规范

## 1. 区域划分与协同

Taro Studio 采用单窗口、多区域协同布局。取消了传统的“写作页”和“预览页”之分。

### 剧本流 (Script Flow)
- 居于界面左侧。
- 提供基于文本的快速输入体验。
- **可缩放结构树**：支持鼠标滚轮或捏合手势进行无缝缩放。
  - 最低缩放级别：仅显示 Group 标题和分支轮廓。
  - 中等缩放级别：显示 Group 内的文本概要。
  - 最高缩放级别：显示并允许编辑所有文本内容和基础设定。

### 舞台画布 (Stage Canvas)
- 居于界面中央。
- 采用真实的游戏分辨率比例渲染。
- 提供拖拽操作支持。例如拖动一个角色立绘，会自动生成或更新当前选中 Group 中的 `move_character` 或 `set_character` 事件绑定，精准记录坐标。

### 动作时间线 (Action Timeline)
- 居于画布下方或右侧。
- 展示选中 Group 内部各个事件绑定的相对时间点。
- 横轴代表进入 Group 后的时间流逝。
- 提供类似视频剪辑软件的体验：拖拽滑块来改变 `timer` 触发器的数值，或对齐不同动作的发生时间。

### 属性面板 / 工具栏 (Inspector & Toolbar)
- 当选中剧本流的文本、画布的图像或时间线的动作块时，右侧属性面板自动展开对应的详细设置。
- 顶部或浮动的工具栏提供 `全局搜索 (Cmd/Ctrl+K)` 快速插入功能。

## 2. 状态联动逻辑

核心原则：**一处选中，处处响应**。
- 当创作者在剧本流中点击 `Group A` 时：
  - 画布立即重绘，通过当前路径上下文推演出抵达 `Group A` 时的所有视觉元素。
  - 时间线刷新，展示 `Group A` 内的绑定事件。
- 当创作者在画布中选中 `角色 B` 的图片时：
  - 时间线中，所有与 `角色 B` 相关的动作块会被高亮。
  - 右侧属性面板展开 `角色 B` 的坐标、缩放、表情选项。

## 3. 键盘快捷键与输入流

为了保持写作的心流：
- **`Enter`**：在文本输入框中默认行为是换行。
- **`Cmd/Ctrl + Enter`**：强制结束当前的 Group（如果是一个简单对话），并在下方创建一个新的空 Group，焦点自动跳入。
- **`/`**：在空行输入斜杠，唤出快捷命令菜单（如 `/choice` 插入选项，`/bg` 切换背景）。
- **`Arrow Up/Down`**：在剧本流的各个块之间顺滑移动光标。

## 4. 路径上下文可见性

在界面的某个固定位置（如面包屑导航栏），始终显示当前的“路径栈”。
例如：`序章 > 岔路口 (选择：左) > 破旧小屋`
创作者可以点击路径栈上的任意节点进行快速回溯，或者在遇到由于分支导致画布状态不确定的情况时，手动强制选择另一条到达当前节点的模拟路径。
