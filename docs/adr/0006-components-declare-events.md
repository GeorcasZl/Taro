# ADR 0006: 组件声明事件而不隐藏流程

状态：已采纳（2026-06-24，取代原「Plugins may recommend flow」）

## 背景
Taro 通过组件系统（HTML/CSS/JS）支持自定义表现和交互。
组件可能封装复杂的流程控制，但这些控制不能对创作者不可见。

## 决策
组件必须声明它能接收的事件（inputs）和能发出的事件（outputs）。
组件可以推荐流程控制方式，但插入的流程控制必须展开为
可见的 Taro 事件绑定。创作者可以查看和编辑展开后的事件。

## 后果
- 好处：创作者始终能理解和修改故事逻辑
- 代价：组件开发者需要额外声明事件接口
