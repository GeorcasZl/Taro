# ADR 0004: 统一编辑界面

状态：已采纳（2026-06-24，取代原「Canvas is not a second source」）

## 背景
原设计将 Writing/Canvas/Inspector/Preview 分离为独立表面。
但创作者在多个面板间跳转体验割裂，且多种叙事模式（RPG/漫画）
需要视觉编辑和文本编辑同时进行。

## 决策
所有编辑功能融合到一个统一界面中。剧本流、舞台画布、时间线、
素材库等区域在同一画面中协同工作。原则保留：所有区域都通过
Document Command 操作同一个 Document，没有隐藏的第二数据源。

## 后果
- 好处：创作体验连贯；减少上下文切换
- 代价：UI 复杂度更高；需要精心的信息层级设计
